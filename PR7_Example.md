# Вариант 0

# Кейс: «Разработка прототипа системы регистрации заявок»

## 📖 Ситуация

В компании **TechFlow** сотрудники ежедневно создают внутренние заявки, связанные с различными рабочими задачами: запросы на техническую поддержку, обслуживание оборудования, доступ к информационным системам и другие организационные вопросы.

На данный момент сотрудники фиксируют заявки вручную в различных файлах и таблицах, что приводит к путанице, дублированию информации и затрудняет контроль выполнения задач.

Для упрощения работы руководство компании решило разработать простое десктоп-приложение, которое позволит сотрудникам регистрировать заявки и отслеживать их статус в единой системе.

Поскольку в будущем планируется автоматизировать обработку заявок с помощью программных роботов, на первом этапе необходимо создать **прототип приложения** и проверить возможность автоматизации базовых операций взаимодействия с его интерфейсом.

В рамках данной практической работы студенту необходимо разработать тестовую версию системы регистрации заявок и реализовать простой сценарий автоматизации работы с ней.

---

# 🧩 Задание 1. Разработка десктоп-приложения

Необходимо разработать простое десктоп-приложение с графическим пользовательским интерфейсом.

Технология разработки выбирается студентом самостоятельно
(например: **Python Tkinter, C# WinForms, Java Swing** или другие средства разработки настольных приложений).

---

## 🖥️ Требования к интерфейсу приложения

Приложение должно содержать форму регистрации заявок.

### 📝 Поля ввода

* **Request Title** — тема заявки
* **Employee** — сотрудник, создавший заявку

### 🎛 Элемент выбора

* **Priority** — приоритет заявки

  * Low
  * Medium
  * High

### 🔘 Кнопки управления

* ➕ **Add Request** — добавление новой заявки
* 🧹 **Clear Form** — очистка формы ввода

---

## 📊 Таблица заявок

В нижней части окна должна располагаться таблица со списком зарегистрированных заявок.

| ID | Title | Employee | Priority | Status |
| -- | ----- | -------- | -------- | ------ |

Статус новой заявки устанавливается автоматически:

```
New
```

При нажатии кнопки **Add Request** введённые данные должны добавляться в таблицу в виде новой строки.

---

# 🤖 Задание 2. Базовая автоматизация приложения

После разработки приложения необходимо создать RPA-процесс в среде **Puzzle RPA**, который автоматизирует действия пользователя.

Робот должен выполнить следующий сценарий:

1️⃣ Запустить разработанное приложение.  
2️⃣ Дождаться появления главного окна программы.  
3️⃣ Последовательно заполнить форму создания заявки:  

* ввести тему заявки
* указать сотрудника
* выбрать приоритет

4️⃣ Добавить в систему **не менее пяти заявок**.  
5️⃣ Проверить появление новых записей в таблице приложения.

---

* ⌨️ ввод значения в элемент
* 🖱 клик по элементу интерфейса
* 🔍 чтение значения элемента
